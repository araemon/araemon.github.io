iPhoneからC言語を使ってUDP通信してみた

2018-11-28

iOS -> Mac with UDP



iOSからMacへUDP送信してみる。
学習がてらNSStreamを使わずに、あえてC言語で書いてみることにする。
UDPはUser Datagram Protocolの略だ。データグラムというのは、配送が成功したかどうかの確認はしないので届くかどうか保証されないですよ、という意味のようだ。


Mac側でUDPサーバーを立ち上げる

まずは、Macのターミナルのnetcatコマンドを使ってさくっとUDPサーバーを立ち上げる。
UDPで待ち受けるにはたったのこれでおk。
nc -u -l 8888

ところで、ポートが解放されているかどうかの確認をしたかったのだが、そもそもUDPは一方的な送信なので、ポートが開いているかどうかのスキャンはできない。
（ポート解放確認のコマンド）
nc -vz 192.168.100.101 8888

次のように、TCPでサーバーを立ち上げるとポートスキャンできる。
nc -l 8888


iOSプログラミング、UDPクライアントを作る

iOS側のプログラミングは以下のようになった。
ViewController.mm

#import "ViewController.h"
#include 
#include 
#include 

@interface ViewController ()

@end

@implementation ViewController

- (void)viewDidLoad {
    [super viewDidLoad];
}

- (IBAction)clickedSendButton:(id)sender {
    int sock;
    struct sockaddr_in addr;
    
    sock = socket(AF_INET, SOCK_DGRAM, 0);
    
    addr.sin_family = AF_INET;
    addr.sin_port = htons(8888);
    addr.sin_addr.s_addr = inet_addr("192.168.100.101");
    
    sendto(sock, "HELLO", 5, 0, (struct sockaddr *)&addr, sizeof(addr));
    
    close(sock);
}
@end

Objective-CでC言語の関数を使いたい場合は`.m`ファイルを`.mm`の拡張子にすれば良いようだ。
sockaddr_inという構造体に送信先アドレスやポート番号を設定してる。
socketという関数を使って、UDPならSOCK_DGRAMを指定してやれば良いようだ。
AF_INETはIPv4のインターネットプロトコルで通信しまっせということらしい。
これでアプリのボタンを押せば、Mac側で立ち上げたターミナル画面にメッセージが表示される。

UDPのプロトコル仕様を確認してみる


ポートが16ビットの長さなので、2の16乗 = 65536番まで指定できるということだ。

Mac側でUDPクライアント使用する

MacからiPhoneへUDPソケットを送ってみる。
Macのターミナルから下記コマンドを入力。
nc -u 192.168.100.119 8888

送信したいメッセージを入力して、リターンを押せばiPhone画面にメッセージが表示される。
